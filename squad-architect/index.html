TUTORIAL 01

What Is Squad Architect?

10 minBeginner

What You'll Learn

  • What Squad Architect does and why it exists
  • The two-phase workflow: Design → Build
  • What gets generated and how squads run

Context

Squad Architect (by Endorphin AI) is a compiled Go CLI that designs, builds, and manages multi-agent squads for Claude Code. Instead of hand-crafting dozens of agent files, skill packages, and pipeline configs, you describe what you need and the CLI generates everything.

Walkthrough

Step 1: The Two-Phase Workflow

Core Workflow
squad-architect design    →  Interactive design with Claude
                              ↓ saves .squad/design.json
squad-architect build     →  Non-interactive build from design
                              ↓ creates agents, skills, commands, pipeline
/el-capitan "task"        →  Squad runs autonomously in Claude Code

Step 2: What Gets Generated

A single design + build cycle creates the entire .claude/ ecosystem:

Generated Structure
.claude/
├── agents/                    # Agent definitions (.md)
│   ├── pm-rn.md
│   ├── backend-rn.md
│   └── ...
├── agent-memory/              # Persistent memory + passports
│   ├── pm-rn/
│   │   ├── MEMORY.md
│   │   └── PASSPORT.md       # Unique ID (pm-rn-agent-001)
│   └── ...
├── skills/                    # Full skill packages
│   ├── pm-rn/
│   │   ├── SKILL.md
│   │   ├── FORMAT.md
│   │   ├── VOICE.md
│   │   ├── DRY_RUN.md
│   │   └── workflows/
│   └── ...
├── commands/
│   ├── el-capitan.md          # Orchestrator (auto-generated)
│   └── invoke-el-capitan.md   # Worker endpoint (auto-generated)
├── workflows/
│   └── rn.yaml                # Pipeline definition
└── CLAUDE.md                  # Project documentation

Step 3: The Factory Principle

Squad Architect builds the factory, not the products. Templates generate everything:

TemplateGenerates
AGENT_TEMPLATE.mdAgent definitions with modes, tasks, rules
SKILL_TEMPLATE.mdSkill SKILL.md + FORMAT.md + VOICE.md
PASSPORT_TEMPLATE.mdUnique agent IDs
EL_CAPITAN_TEMPLATE.mdOrchestrator command
PIPELINE_TEMPLATE.mdWorkflow YAML with phases and gates

Key Takeaways

  • Squad Architect = CLI that generates complete agent squads for Claude Code
  • Two phases: Design (interactive) → Build (non-interactive)
  • Generates agents, skills, commands, pipeline YAML, orchestrator, CLAUDE.md
  • Factory principle: templates generate everything, fix the factory not the output
TUTORIAL 02

Installation

10 minBeginner

What You'll Learn

  • Install Squad Architect via quick install or manual download
  • Prerequisites: Claude Code CLI
  • Verify the installation

Walkthrough

Step 1: Prerequisites

Squad Architect requires Claude Code CLI installed and authenticated:

Terminal
# Install Claude Code CLI
npm install -g @anthropic-ai/claude-code

# Verify Claude is authenticated
claude --version

Step 2: Quick Install (macOS / Linux)

Terminal
# One-line install (detects OS + architecture automatically)
curl -fsSL https://raw.githubusercontent.com/endorphin-ai/squad-architect/main/install.sh | bash

The installer detects your OS (darwin/linux) and architecture (amd64/arm64), downloads the latest release with checksum verification, and places the binary in your PATH.

Step 3: Manual Download

Download from GitHub Releases:

Terminal
# Download the binary for your platform
# macOS M-series: squad-architect_*_darwin_arm64.tar.gz
# macOS Intel:    squad-architect_*_darwin_amd64.tar.gz
# Linux x86_64:   squad-architect_*_linux_amd64.tar.gz

tar -xzf squad-architect_*.tar.gz
sudo mv squad-architect /usr/local/bin/

Step 4: Verify Installation

Terminal
squad-architect --help

Key Takeaways

  • Requires Claude Code CLI installed and authenticated
  • Quick install via curl | bash with checksum verification
  • Manual download from GitHub Releases for all platforms

Design — Interactive Squad Blueprint

20 minBeginner

What You'll Learn

  • How to run squad-architect design
  • Design session slash commands: /commands, /skills, /save, /cancel
  • How to approve a design
  • Design resume: pick up where you left off

Walkthrough

Step 1: Start a Design Session

Terminal
cd your-project
squad-architect design

The CLI scans your project, loads embedded catalogs, and opens an interactive session with Claude. Describe the squad you need:

Design Session
╭──────────────────────────────────────────╮
│  Squad Architect — Design Session        │
│  Scanning project...                     │
╰──────────────────────────────────────────╯

ℹ Found: 3 agents, 4 skills, 2 commands

Describe the squad you want to build:
❯ I need a fullstack squad for React Native + Express API

◐ Designing squad...

I'll design a 6-agent squad for your React Native + Express stack:

| Agent            | Role              | Skill           | Phase | Model  |
|------------------|-------------------|-----------------|-------|--------|
| pm-rn            | Product Manager   | pm-rn           | 0     | sonnet |
| api-architect-rn | API Architect     | api-design-rn   | 1     | opus   |
| backend-rn       | Express Backend   | express-backend | 2     | opus   |
| frontend-rn      | React Native      | rn-frontend     | 3     | sonnet |
| qa-rn            | QA Engineer       | testing-rn      | 4     | sonnet |
| reviewer-rn      | Code Reviewer     | code-review-rn  | 5     | opus   |

Type `/commands` to pick commands, `/skills` to pick skills.

Step 2: Slash Commands During Design

CommandWhat it does
/commandsOpen TUI picker — browse and select embedded commands
/skillsOpen TUI picker — browse and select embedded skills
/saveSave conversation draft (not final design)
/restartClear history, start fresh
/historyShow turn count
/cancelExit without saving

Step 3: Approve the Design

When you're satisfied, use any approval phrase:

Approval
❯ approve

◐ Generating design...
✓ Design saved to .squad/design.json
✓ 6 agents, 6 skills, 5 phases defined

Next: squad-architect build

Approval phrases: approve, looks good, lgtm, ship it, build it.

Step 4: Design Resume

If .squad/design.json already exists, the CLI detects it and offers:

  • Resume — Continue from the existing design
  • Restart — Start fresh, discard existing design
  • Cancel — Exit
Terminal
# Resume non-interactively
squad-architect design --resume

Step 5: Design Flags

FlagDefaultDescription
--dir.Project directory to scan
--output.squad/design.jsonDesign file output path
--modelfrom configClaude model to use
--resume—Resume from existing design

Try It Yourself

Run squad-architect design in a project and describe a 3-agent squad (PM + Developer + Reviewer). Use /commands to browse available commands. Approve the design and inspect .squad/design.json.

Hint

Start simple: "I need a squad with a PM, a backend developer, and a code reviewer for my Node.js API." Claude will propose agents, skills, and phases. You can iterate before approving.

Key Takeaways

  • squad-architect design opens an interactive session with Claude
  • Use /commands and /skills to browse embedded catalogs
  • Approve with "approve", "lgtm", "ship it", or "build it"
  • Design saves to .squad/design.json — resume later with --resume

Step 2: Build Phases

The build progresses through 6 phases:

#PhaseWhat happens
1ScaffoldingCreates skill folders and agent files from templates
2Filling skillsClaude fills SKILL.md, FORMAT.md, VOICE.md, workflows
3Customizing agentsClaude writes agent-specific modes, tasks, rules
4Generating orchestrationCreates el-capitan, invoke-el-capitan, workflow YAML
5Generating CLAUDE.mdWrites project documentation with squad info
6VerifyingChecks for zero TODOs, validates all files

Step 3: Build Flags

FlagDefaultDescription
--design.squad/design.jsonDesign file path
--dir.Target project directory
--verbose—Show detailed Claude output
--timeout—Build timeout (e.g. 30m, 1h)
--log—Save full build log to file

Step 4: Build Resume

If a build is interrupted, the CLI saves progress to .squad/build-state.json. Next time you run build, it offers to resume:

Terminal
# Resume interrupted build automatically
squad-architect build --resume

Step 5: Agent Passports

Every scaffolded agent gets a unique passport with an auto-incrementing ID:

.claude/agent-memory/backend-rn/PASSPORT.md
---
agent_id: "backend-rn-agent-001"
agent_name: "backend-rn"
squad: "rn"
created: "2026-03-25"
---

Key Takeaways

  • squad-architect build reads design.json and generates everything
  • 6 build phases: scaffold → fill skills → customize agents → orchestration → CLAUDE.md → verify
  • Build state saved to .squad/build-state.json — resume with --resume
  • Every agent gets a unique passport ID

Run — One-Step Squad Creation

15 minIntermediate

What You'll Learn

  • How squad-architect run combines design + build
  • Using the generated /el-capitan command
  • The El-Capitan orchestration pipeline

Walkthrough

Step 1: Combined Design + Build

Terminal
# Interactive design, then auto-build
squad-architect run

# Or provide a description upfront
squad-architect run "mobile app squad with Flutter + Firebase"

Step 2: Using the Generated Squad

After building, open Claude Code in your project and use the orchestrator:

Claude Code
# Run the full pipeline
/el-capitan "build a REST API for user management"

# Dry-run — agents analyze without writing files
/el-capitan --dry-run "refactor the auth module"

Step 3: The Pipeline in Action

El-Capitan Pipeline
/el-capitan "build a todo app"
│
├── Phase 0: Task(pm-rn)          → PRD document
│   [gate: features + criteria defined]
├── Phase 1: Task(api-architect)   → Architecture
│   [gate: interfaces defined]
├── Phase 2: Task(backend-rn)      → Backend code
│   [gate: build passes]
├── Phase 3: Task(frontend-rn)     → Frontend code
│   [gate: build passes]
├── Phase 4: Task(qa-rn)           → Tests
│   [gate: tests pass]
├── Phase 5: Task(reviewer-rn)     → Code review
│   [gate: 0 critical issues]
│
▼
Pipeline Report to User

Star topology: El-capitan dispatches all workers. Workers report back. No worker-to-worker communication.

Key Rule: The orchestrator NEVER does workers' jobs. It dispatches via Task() and collects results. Workers never talk to each other — only to the orchestrator.

Key Takeaways

  • squad-architect run = design + build in one step
  • /el-capitan "task" runs the full pipeline in Claude Code
  • Star topology: orchestrator dispatches, workers report back
  • Each phase has a validation gate — pipeline stops on failure

Audit, Adjust & Insight

20 minAdvanced

What You'll Learn

  • audit — Analyze squad health
  • adjust — Modify squads with natural language
  • insight — Generate intelligence reports

Walkthrough

Step 1: Audit — Squad Health Check

Terminal
# Full squad analysis
squad-architect audit --code=<CODE>

# JSON output
squad-architect audit --code=<CODE> --format json --output audit.json

Audit scans your squad and reports: inventory, coverage, anti-patterns, and recommendations.

Step 2: Adjust — Natural Language Modifications

Terminal
# Add a new agent
squad-architect adjust "add a security scanner agent to the pipeline" --code=<CODE>

# Rename and update references
squad-architect adjust "rename backend-dev to api-dev and update all references" --code=<CODE>

Step 3: Insight — Intelligence Reports

Terminal
# Health overview
squad-architect insight health --code=<CODE>

# Find capability gaps
squad-architect insight gaps --code=<CODE>

# Performance analysis
squad-architect insight performance --code=<CODE>

# Everything combined
squad-architect insight full --code=<CODE> --output full-report.md

Insight Report Types

TypeWhat it analyzes
healthOverall squad health, coverage, coherence
gapsMissing capabilities, uncovered domains
performanceAgent efficiency, pipeline bottlenecks
composeComposition quality, inter-agent dependencies
fullAll of the above combined
Access Control: audit, adjust, and insight are gated commands requiring --code=<CODE>. Core commands (design, build, run, review) are ungated.

Key Takeaways

  • audit — comprehensive health check of your squad
  • adjust — modify squads with natural language instructions
  • insight — 5 report types: health, gaps, performance, compose, full
  • All three are gated admin commands (require access code)
TUTORIAL 08

Full Workflow — End to End

15 minAdvanced

What You'll Learn

  • Complete workflow from zero to running squad
  • All commands and global flags at a glance
  • Tips for production use

Walkthrough

Step 1: The Complete Workflow

Terminal
# 1. Design your squad (interactive)
squad-architect design
# → Describe your squad, iterate, approve

# 2. Build the squad (non-interactive)
squad-architect build
# → Scaffolds everything from design.json

# 3. Use the squad in Claude Code
cd your-project
/el-capitan "build a REST API for user management"
# → Pipeline runs: PM → Architect → Dev → Tests → Review

# 4. Review quality
squad-architect review agent .claude/agents/backend-rn.md

# 5. Audit squad health
squad-architect audit --code=<CODE>

# 6. Adjust from audit findings
squad-architect adjust "add validation to skill X" --code=<CODE>

# 7. Generate insights
squad-architect insight full --code=<CODE> --output report.md

Step 2: Command Reference

TierCommandsAccess
Ungateddesign, build, run, reviewNo code required
Gatedaudit, adjust, insight, version, sync, selectRequires --code

Step 3: Global Flags

FlagDescription
--claude-pathPath to Claude CLI binary (default: auto-detect)
--codeAccess code for gated commands
--debugEnable debug logging
--quietMinimal output
--no-colorDisable colors (accessibility)

Step 4: Production Tips

  • Start with design, not run — review the design before building
  • Use --verbose on first build to see what Claude is doing
  • Run review on generated agents before using the squad
  • Run audit periodically to check squad health
  • Use adjust for incremental changes — don't rebuild from scratch
  • Save build logs with --log build.log for debugging
